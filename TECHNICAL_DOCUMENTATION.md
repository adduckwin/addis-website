# Техническая документация проекта Addis Coffee

## 1. Общие сведения

**Название проекта:** Addis Coffee  
**Тип:** Интернет-магазин кофе (B2C + B2B)  
**Стек технологий:** React, TypeScript, Vite, Tailwind CSS, Zustand, Shadcn/UI  
**Интеграции:** СБИС (API), DaData (проверка контрагентов), 1С (через промежуточный слой)  
**Целевая аудитория:** Розничные покупатели, оптовые клиенты (кофейни, офисы), дистрибьюторы.

### 2.2. State Management (Zustand)
Приложение использует глобальное состояние для:
- **Корзины (`cartStore`):** Добавление товаров, расчет скидок (розница/опт), применение промокодов.
- **Дизайна (`designStore`):** Динамическое изменение цветов, шрифтов, логотипов через админку.
- **Пользователя:** Статус авторизации, роль (client/wholesale/admin).

## 3. Функциональные модули

### 3.1. Публичная часть (Frontend)

#### Главная страница (`/`)
- **Hero-блок:** Слайдер с акциями, кнопка "В каталог".
- **Категории:** Быстрый переход по типам кофе (Зерновой, Молотый, Капсулы).
- **Хиты продаж:** Карусель популярных товаров.
- **Опт:** Блок преимуществ для B2B с формой быстрой заявки.
- **Блог:** Превью последних статей.

#### Каталог и Товары
- **Фильтрация:** По цене, обжарке, стране происхождения, вкусу.
- **Карточка товара:** Фото, описание, выбор фасовки, кнопка "В корзину", блок "С этим покупают".
- **Умный поиск:** Поиск по названию и характеристикам.

#### Корзина и Оформление заказа
- **Розничный поток:** Быстрое оформление, выбор доставки (СДЭК, Почта), оплата онлайн.
- **Оптовый поток:**
  - Проверка статуса компании (по ИНН).
  - Генерация счета на оплату.
  - Отправка заказа в СБИС.

#### Личный кабинет
- **Покупатель:** История заказов, статусы, баллы лояльности.
- **Оптовик:** Специальные цены, документы (счета, накладные), повтор заказа.

### 3.2. Административная панель (`/admin`)

#### Управление контентом
- Редактирование товаров (CRUD).
- Управление статьями блога.
- Настройка баннеров на главной.

#### Управление дизайном (`/admin/design`)
- **Цветовая схема:** Изменение основных цветов (Primary, Secondary, Accent) с предпросмотром.
- **Типографика:** Выбор шрифтовых пар.
- **Логотипы:** Загрузка SVG/PNG для хедера и футера.
- *Техническая реализация:* Изменения сохраняются в `localStorage` (для демо) или отправляются на бэкенд, применяются через CSS-переменные (`:root`).

#### Интеграции
- Настройка подключения к СБИС (токен, URL).
- Тест соединения с внешними API.

## 4. Интеграция с внешними сервисами

### 4.1. Интеграция со СБИС (SBIS API)

#### 4.1.1. Описание коннектора
Файл: `src/lib/sbis-api.ts`  
Реализует взаимодействие с экосистемой СБИС (Тензор) для автоматизации B2B-процессов.

#### 4.1.2. Доступные методы

| Метод | Описание | Входные данные | Ответ |
|-------|----------|----------------|-------|
| `auth()` | Получение токена доступа | `appId`, `appSecret` | `access_token` |
| `getCounterpartyByInn(inn)` | Поиск контрагента по ИНН | `inn` (string) | Объект компании (название, адрес, статус) |
| `createOrder(data)` | Создание заказа в СБИС | Товары, Клиент, Цены | `orderId`, статус |
| `getPriceList()` | Выгрузка прайс-листа | Фильтры категорий | Массив товаров с ценами и остатками |
| `updateStatus(orderId, status)` | Обновление статуса заказа | `orderId`, `status` | Результат обновления |

#### 4.1.3. Логика работы

##### Синхронизация товаров
1. Сайт запрашивает актуальный прайс из СБИС (через прокси-сервер).
2. Обновляет локальную базу или кэш товаров.
3. **Важно:** Прямой вызов из браузера запрещен политикой CORS, требуется сервер-посредник (Node.js/PHP/Python).

##### B2B Регистрация
1. Пользователь вводит ИНН.
2. Сайт вызывает `getCounterpartyByInn`.
3. Данные подставляются в форму регистрации.
4. При одобрении менеджером пользователю присваивается роль `wholesale`.

##### Оформление заказа
1. При оформлении оптового заказа данные уходят в СБИС как "Заказ покупателя".
2. В ответ возвращается номер документа для отслеживания.

#### 4.1.4. Требования к безопасности
- **API ключи** не должны храниться в коде фронтенда.
- Все запросы к СБИС должны идти через защищенный бэкенд (`/api/proxy/sbis`).
- Использование HTTPS обязательно.

#### 4.1.5. Архитектура взаимодействия
```
[Frontend] <--HTTPS/JSON--> [Backend Proxy] <--API SBIS--> [СБИС Облако]
```
- **Frontend:** Отправляет запросы на свой бэкенд.
- **Backend Proxy:** Хранит секретные ключи, формирует подписанные запросы к СБИС.
- **СБИС:** Обрабатывает запросы, возвращает данные.

### 4.2. Интеграция с DaData (Проверка контрагентов)

#### 4.2.1. Описание
Сервис для автоматического заполнения реквизитов юридических лиц по ИНН. Используется при регистрации оптовых клиентов.

#### 4.2.2. Технические детали
- **Endpoint:** `https://suggestions.dadata.ru/suggestions/api/4_1/rs/findById/party`
- **Метод:** `POST`
- **Заголовки:**
  - `Content-Type: application/json`
  - `Accept: application/json`
  - `Authorization: Token {API_KEY}`
- **Тело запроса:**
  ```json
  { "query": "7707083893" }
  ```

#### 4.2.3. Реализация на сайте
1. **Серверный прокси:** Создан маршрут `/api/dadata/fetch`, который скрывает API-ключ от клиента.
   - Принимает INN от фронтенда.
   - Делает запрос к DaData.
   - Возвращает структурированные данные: название, адрес, руководитель, контакты, КПП, ОГРН.
   
2. **Клиентский хук:** `useDaDataParty`
   - Автоматически отправляет запрос при вводе ИНН (с задержкой/debounce).
   - Управляет состояниями загрузки и ошибок.
   - Заполняет поля формы полученными данными.

#### 4.2.4. Сценарий использования
1. Менеджер открывает форму регистрации оптового клиента.
2. Вводит ИНН организации (например, `7707083893`).
3. Система автоматически подтягивает:
   - Полное наименование компании.
   - Юридический адрес.
   - ФИО руководителя.
   - Контактные телефоны/email.
   - КПП и ОГРН (скрытые поля).
4. Пользователь проверяет данные и завершает регистрацию.

#### 4.2.5. Настройка
- API-ключ вводится в админ-панели в разделе "Настройки интеграций".
- Требуется действующая подписка на тарифе DaData "Стандарт" или выше для безлимитных запросов.

## 7. Технические требования и развертывание

### 7.1. Переменные окружения (.env)
```bash
# API Endpoints
VITE_SBIS_API_URL=https://api.sbistenant.ru
VITE_DADATA_API_KEY=ваш_ключ_dadata
VITE_SERVER_PROXY_URL=http://localhost:3001/api

# Настройки
VITE_APP_NAME="Addis Coffee"
VITE_SUPPORT_EMAIL=support@addis-coffee.ru

# Email (SMTP)
VITE_SMTP_HOST=smtp.yandex.ru
VITE_SMTP_PORT=465
VITE_SMTP_USER=your@email.com
VITE_SMTP_PASS=app_password

# SMS (опционально)
VITE_SMS_PROVIDER=smsru
VITE_SMS_API_KEY=your_sms_api_key
```

### 7.2. Сборка и запуск
```bash
# Установка зависимостей
npm install

# Запуск разработки
npm run dev

# Продакшн сборка
npm run build
npm run preview
```

### 7.3. Серверная часть (Рекомендации)
Для полноценной работы интеграций необходим backend (Node.js Express/NestJS или PHP Laravel):

1. **Прокси для СБИС:**
   - Хранит секреты (AppID, Secret)
   - Делает запросы к тензору
   - Отдает данные фронтенду

2. **База данных:**
   - PostgreSQL или MySQL
   - Хранение пользователей, заказов, кэша товаров

3. **Очереди:**
   - Redis/Bull для фоновой синхронизации цен

## 8. Безопасность и соответствие РКН

### 8.1. Соответствие требованиям РФ
- **Отсутствие блокируемых ресурсов:** Удалены все ссылки и зависимости от сервисов Manus, Kimi, Instagram, Facebook.
- **Локализация:** Рекомендовано размещение серверов на территории РФ.
- **152-ФЗ:** Формы содержат чекбоксы согласия на обработку персональных данных.

### 8.2. Защита данных
- Валидация всех входящих данных.
- Санитизация HTML в блоге.
- Защита от XSS и CSRF (на уровне бэкенда).
- Хранение токенов СБИС только на сервере.
